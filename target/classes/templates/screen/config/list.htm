• server:
    • port: 端口号，没有默认值，必要配置项
    • context-path: 执行路径，没有默认值，必要配置项
    • domain: application的域名，没有默认值，必要配置项
  • Melody:
    • connection:
      • maxTotal: 最大连接数，默认值200，非必要配置项
      • maxPreRoute: 最大路由数，默认值50，非必要配置项
      • socketTimeOut: socket超时时间，单位毫秒，默认值5000，非必要配置项
      • connectionTimeOut: 连接超时时间，单位毫秒，默认值3000，非必要配置项
      • connectionRequestTimeOut: request超时时间，单位毫秒，默认值3000，非必要配置项
    • web:
      • meta:
        • cookie:
          • name: 存储的cookie名，默认为"_h_"，非必要配置项
          • maxCount: cookie存储对象的最大个数，默认为20，非必要配置项
          • maxLength: cookie存储对象的最大长度，单位为字节，默认为4095，非必要配置项
          • domain: cookie的域，默认为localhost，非必要配置项
          • path:
      • encoding: 字符编码，默认UTF-8，非必要配置项
      • monitor:
        • threshold: 性能监控配置项，响应时间超过改配置值的访问会被记录，默认值100，非必要配置项
      • locale:
        • cookieName: 存储国际化信息的cookie名，默认为"_l_"，非必要配置项
        • support: 支持的语言，默认为zh_CN，非必要配置项
        • default: 默认的语言，默认为zh_CN，非必要配置项
        • country: 语言格式中是否带有国家，默认为false，非必要配置项
        • language: 语言格式中是否带有语言类别，默认为true，非必要配置项
  • spring:
    • thymeleaf:
      • cache: 是否开启页面缓存，默认为false，非必要配置项
      • prefix: 页面文件根目录，默认为"classpath:/views/"，非必要配置项
      • suffix: 页面文件后缀，默认为.htm，非必要配置项
      • encoding: 页面字符编码，默认为UTF-8，非必要配置项
      • content-type: 内容类型，默认为text/html，非必要配置项
      • mode: 模板类型，默认为html5，非必要配置项
    • data:
      • mongodb:
        • uri: mongodb的地址，没有默认值，如果使用mongodb，则为必要配置项
    • redis:
      • database:
      • host: redis的主机地址，使用redis的必要配置项
      • port: redis的主机地址的端口，使用redis的必要配置项
      • password: redis的访问密码，使用redis的必要配置项
      • pool:
        • max-active: 连接池的最大连接数，使用redis的必要配置项
        • max-wait: 连接的最大等待时间，使用redis的必要配置项项
        • max-idle: 连接池的最大空闲连接，使用redis的必要配置项
        • min-idle: 连接池的最小空闲链接，使用redis的必要配置项
      • timeout: 连接超时时间，使用redis的必要配置项
    • datasource:
      • password: 数据库访问密码，必要配置项
      • username: 数据库用户名，必要配置项
      • url: 数据库连接地址，必要配置项
      • driver-class-name: 数据库驱动名，必要配置项
      • type:
      • druid:
        • initial-size: 初始化连接数
        • max-active: 最大激活数
        • max-wait: 最大等待时间
        • use-unfair-lock: 是否启动非公平锁
        • pool-prepared-statements: mysql 用 false | oracle 用 true
        • filters: 启动的监听、日志、防注入的功能
        • validation-query: 验证的sql语句
        • validation-query-timeout: 验证超时时间
        • test-on-borrow: 连接时是否开启验证
        • test-on-return: 退还连接时是否开启验证
        • test-while-idle: 对r空闲连接是否开启验证（建议配置为tre，不影响性能，并且保证安全性 ）
        • time-between-eviction-runs-millis: Destroy线程会检测连接的间隔时间 || testWhileIdle的判断依据
  • logging:
    • level:
      • root: 日志等级
